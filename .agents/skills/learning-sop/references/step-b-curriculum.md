# Step B：按需生成课程

## AI 角色

你是 **{topic}** 领域的顶尖专家和课程设计师。你已了解学习者当前水平（来自 Step A），现在需要生成一份**精准针对其盲区**的课程，而非重复已掌握的内容。

---

## 输入

读取 `cycles/cycle{N}/diagnosis.md`，获取：
- `knowledge_level`：当前水平
- `blind_spots`：本轮重点攻克的盲区
- `known_concepts`：已知概念（课程中直接引用，不需重新解释）
- `learning_context`：学习动机/应用场景（用于选取贴近的例子）

---

## 课程难度映射

| `knowledge_level` | 课程定位 | 内容侧重 |
|-------------------|---------|---------|
| 0-2 | 入门 | 核心概念建立、直觉理解、大量类比 |
| 3-4 | 基础 | 原理推导、常见应用、典型案例 |
| 5-6 | 进阶 | 边界条件、例外情况、模型局限 |
| 7-8 | 深入 | 跨领域联系、前沿争议、批判视角 |
| 9-10 | 专家 | 原创思考、研究方向、未解问题 |

---

## 课程生成 Prompt

```
你是 {topic} 领域专家。

学习者当前水平：{level}/10（{level_description}）
已掌握的概念：{known_concepts}（这些内容勿重复讲解，可直接引用）
本轮需攻克的盲区：{blind_spots}
学习者背景/动机：{learning_context}

请生成一份专门针对上述盲区的课程，要求：
1. **从已知出发**：用学习者已掌握的概念作为锚点，建立新知识连接
2. **聚焦盲区**：每个模块对应一个盲区，不发散
3. **自包含**：课程本身包含理解所需的全部内容，无需查阅外部资料
4. **例子贴近背景**：根据学习者的应用场景选取例子
5. **预埋练习题**：每个模块末尾给出 1-2 道练习题（Step C 使用）
```

---

## 输出格式

```markdown
# 第 {cycle} 轮课程：{topic}
**难度级别**：{level_name}（{level}/10）
**本轮攻克目标**：{blind_spots 列表}

---

## 模块一：[对应盲区1的标题]

### 从你知道的说起
[用 known_concepts 中的概念作为引入，1-2 句]

### 核心内容
[详细解释，400-600 字，使用类比和具体例子]

### 关键要点
- 要点1
- 要点2
- 要点3

### 练习题
1. [题目1]（答案在 Step C 中揭晓）
2. [题目2]

---

## 模块二：[对应盲区2的标题]
[同上结构]

---

## 综合理解检验
[一个串联本轮所有模块的综合场景题，Step C 迁移练习使用]
```

---

## 输出文件

保存到 `cycles/cycle{N}/curriculum.md`，带 frontmatter：

```markdown
---
cycle: {N}
knowledge_level_at_generation: {level}
blind_spots_addressed:
  - {blind_spot_1}
  - {blind_spot_2}
generated_at: {timestamp}
---

{完整课程内容}
```

---

## 课程生成完毕后

告知用户：

```
课程已准备好，请阅读上方内容。
读完后告诉我，我们进入互动练习环节（约 40 分钟）。
有不清楚的地方可以先提问，没有的话直接说"读完了"即可。
```
