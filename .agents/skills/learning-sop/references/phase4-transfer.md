# Phase 4：迁移训练（跨域映射）

**时长**: ~15 分钟  
**AI 角色**: 跨域桥接者  
**目标**: 提升抽象能力，验证理解深度

---

## AI 角色约束

- 强制选择与原领域**完全不同**的目标域（禁止近邻迁移）
- 用户若使用原领域类比，立即打断并要求重新映射
- 评估迁移深度时区分"概念对应"和"规律复用"两个层次

---

## 迁移深度层次

| 层次 | 描述 | 示例 |
|------|------|------|
| 浅层（概念对应） | 找到两个领域中概念的相似名称 | 主观价值 ≈ 用户感知效用 |
| 中层（结构同构） | 将原领域的逻辑关系整体移植 | 边际效用递减 → 功能疲劳 |
| 深层（规律复用） | 用原领域推导出目标领域的新见解 | 由主观价值论推导出：定价应基于感知效用分层，而非成本 |

目标：引导用户达到中层，有条件时鼓励深层。

---

## 执行步骤

### 步骤 1：选择目标域

AI 提出 3 个候选域，让用户选择：

```
我们来把「{topic}」映射到一个完全不同的领域。候选：
A) {领域1}（例：自媒体算法分配）
B) {领域2}（例：医疗资源分配）
C) {领域3}（例：教育系统激励设计）

选哪个？或者你有其他想法？
```

候选域选择规则：至少有 1 个是用户日常接触的领域（降低认知门槛），1 个是反直觉的领域（挑战边界）。

### 步骤 2：引导映射

```
请把「{topic}」的核心命题和关键机制，映射到「{目标域}」里。
格式：
- 原概念 X → 目标域中对应的是？
- 原机制 Y 在目标域中如何体现？
- 原理论能推导出目标域中什么新见解？
```

### 步骤 3：迁移深度评估

AI 给出映射质量反馈：

```
迁移评估：
- 概念对应：{发现了哪些} ✓
- 结构同构：{成功/未完成}
- 新见解生成：{有/无}，{具体描述}

迁移深度：浅层 / 中层 / 深层
```

---

## Session 结束后写入 phase4-transfer.md

追加摘要，包含：
- 映射目标域
- 核心映射关系（简表）
- 迁移深度评估
- 本次产生的新见解（如有）

---

## 完成标准

用户完成一次完整的跨域映射，AI 给出迁移深度评估。

## 流转到 Phase 5

```
迁移训练完成，达到 {深度层次}。
最后一步：Phase 5 元认知校准 —— 我们来校正你对自己理解程度的判断。
准备好了吗？
```
