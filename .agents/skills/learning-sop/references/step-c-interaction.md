# Step C：学习互动

## 前提

学习者已阅读 `cycles/cycle{N}/curriculum.md` 中的课程内容。
进入互动前确认："课程读完了吗？准备好了告诉我，我们开始互动练习。"

---

## C1：费曼测试（~15 min）

### AI 角色

苏格拉底式提问者，通过追问暴露理解漏洞，不直接给答案。

### 执行步骤

1. 从课程中选取**最核心的 1 个概念**（优先选 `blind_spots` 排第一位的）
2. 发出费曼挑战：

```
不看课程，用你自己的话解释：{核心概念}
假设你在向一个聪明但完全不懂这个领域的朋友解释。
```

3. 按以下标准评估并追问：

| 回答质量 | 追问策略 |
|---------|---------|
| 解释准确 | 追问边界："那如果 [极端场景]，这个解释还成立吗？" |
| 解释模糊 | 要求具体化："能给我一个具体的例子吗？" |
| 概念混淆 | 澄清反问："你说的 [X] 和 [Y] 在你理解里是什么关系？" |
| 解释错误 | 不直接纠正，给出反例："如果你的解释成立，那 [反例] 应该怎么解释？" |

4. 费曼测试结束后给出反馈：

```markdown
## 费曼测试反馈
- 理解准确的部分：[...]
- 需要补充/修正的地方：[...]
- 费曼得分：{X}/10
```

---

## C2：检索练习（~15 min）

### AI 角色

出题官，逐题展开，作答错误时引导推理而非直接给答案。

### 执行步骤

1. 读取课程 `cycles/cycle{N}/curriculum.md`，**了解本轮覆盖的知识点和盲区**，但**不得直接使用课程末尾的练习题**（学习者在读课程时已见过这些题，直接使用会降低检索难度，无法真实评估掌握程度）。基于相同知识点**重新设计**题目：换场景、换问法、换方向，确保题目对学习者是全新的。
2. 等待作答后给出反馈：

**答对时：**
```
✓ 正确！补充一个延伸理解：[...]
```

**答错/不完整时：**
```
还不太对，给你一个提示：
[提示：引导推理方向，但不给出答案]

再想想？
```
若第二次仍错：
```
核心在于：[解释正确答案的推理过程，而非直接给结论]
```

3. 全部题目完成后汇总，记录每题作答到 `cycles/cycle{N}/quiz.md`：

```markdown
## 检索练习小结（Cycle {N}）
- 完全正确：{X} 题
- 提示后答对：{Y} 题
- 理解有偏差：{Z} 题
- 正确率：{X/(X+Y+Z)*100:.0f}%

| # | 题目摘要 | 结果 | 备注 |
|---|---------|------|------|
| 1 | [题目摘要] | ✓ / △ / ✗ | [错误点] |
```

---

## C3：迁移应用（~10 min）

### AI 角色

场景设计师，给出课程中**未直接出现**的新场景，考察知识迁移能力。

### 执行步骤

1. 读取诊断中的 `learning_context`，设计贴近用户实际背景但课程中未涉及的新场景。**不得使用课程末尾的"综合理解检验"题**（学习者在读课程时已见过），必须重新设计场景：

```
## 迁移练习

场景：[新场景描述，100-200 字，包含足够背景信息]

问题：请用本轮学到的知识分析这个场景，你会怎么思考？
```

2. 评估迁移质量并点评：

| 迁移质量 | 描述 | 分值 |
|---------|------|------|
| 表面迁移 | 套用词汇，没有真正理解 | 1-3 |
| 正确迁移 | 核心原理应用正确，但有遗漏 | 4-6 |
| 深度迁移 | 准确应用并能识别场景特殊性 | 7-8 |
| 创造性迁移 | 发现课程中未提及的新联系 | 9-10 |

```markdown
## 迁移应用点评
[具体点评：指出做得好的迁移路径和可以改进的地方]
迁移质量得分：{X}/10
```
