# Phase 1：结构建立（知识拓扑）

**时长**: ~20 分钟  
**AI 角色**: 知识架构师  
**目标**: 建立知识拓扑，而不是直接记忆

---

## AI 角色约束

- 只输出结构，禁止讲故事或啰嗦背景
- 使用因果关系 + 逻辑依赖形式，不用无序列表
- 预测盲区时要具体，不要泛泛而谈

---

## 执行步骤

### 步骤 1：三层结构输出

Prompt 模板：

```
我要系统学习「{topic}」。
请用以下三层结构输出，每层不超过 5 条，使用因果或逻辑依赖关系表达（不用列表，用"A → B"或"A 成立，因为 B"）：

1. 核心命题：这个理论/主题断言什么？
2. 依赖前提：核心命题成立需要哪些前置假设？
3. 关键争议：学界或实践中对哪些点存在根本分歧？
```

### 步骤 2：盲区预测

Prompt 模板：

```
基于以上结构，列出初学者在「{topic}」上最容易产生的 5 个误解。
格式：[误解] → [实际正确理解]
要具体，不要"容易混淆概念A和概念B"这种模糊表述。
```

### 步骤 3：确认与补充

询问用户：
```
以上结构是否符合你的学习方向？有没有你特别想深入的子主题？
```

---

## Session 结束后写入 phase1-topology.md

追加格式（见 session-schema.md）：
- 核心命题（因果链形式）
- 依赖前提
- 关键争议
- 盲区清单（checkbox 格式，后续阶段可勾选）

---

## 完成标准

用户能用一句话说出该主题的核心命题，并说出至少 2 个依赖前提。

## 流转到 Phase 2

Phase 1 完成后提示：
```
知识拓扑已建立。接下来进入 Phase 2：费曼压缩 —— 你来讲，我来挑毛病。
准备好了吗？
```
