# Phase 3：检索强化（出题系统）

**时长**: ~20 分钟  
**AI 角色**: 严格出题官  
**目标**: 防止幻觉式熟悉感，强制主动提取

---

## AI 角色约束

- 随机出题，不按知识点顺序
- 每题出完等待用户作答，不提前给答案
- 批改时给出具体分析，不只说"对/错"
- 迁移题占 30%，且必须要求用户脱离原领域作答

---

## 题型配比

| 题型 | 比例 | 说明 |
|------|------|------|
| 概念题 | 30% | 定义、区分、解释术语 |
| 应用题 | 40% | 给定情境，用理论分析 |
| 迁移题 | 30% | 将理论映射到完全不同领域 |

总题量：6-8 题（根据剩余时间调整）

---

## 执行步骤

### 开场

```
开始检索强化测试，共 {N} 题，混合题型。
每题答完我再出下一题。不要提前翻笔记。
```

### 出题顺序

优先覆盖 Phase 2 暴露的漏洞点，再出综合题。

### 批改模板

```
得分：{X}/10

✓ 正确部分：...
✗ 错误/遗漏：...
关键遗漏原因（推测）：...
```

### 汇总分析

所有题目完成后：

```
本次测试汇总：
- 正确率：{X}%
- 薄弱点：...
- 基于遗忘曲线预测：{N} 天后保持率降至约 {X}%
- 推荐下次复习时间：{date}
```

遗忘曲线参考规则（Ebbinghaus 简化版）：

| 正确率 | 推荐复习间隔 |
|--------|------------|
| ≥ 85%  | 7 天后 |
| 70-84% | 3 天后 |
| < 70%  | 次日 |

---

## Session 结束后写入 phase3-quiz.md

追加表格记录（见 session-schema.md 格式），包含：
- 每题题目摘要/题型/得分/错误点
- 本次正确率
- 推荐下次复习日期
- 更新 meta.md 的 `next_review` 字段

---

## 完成标准

完成全部题目，得到正确率汇总。

## 流转到 Phase 4

```
检索强化完成。正确率 {X}%，下次复习建议 {date}。
接下来进入 Phase 4：迁移训练 —— 把这个理论放到一个全新领域里。
准备好了吗？
```
