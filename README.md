# 大齐 AI 实验室

大齐 AI 实验室是一个以 AI 实战内容为核心的自媒体项目。

这个仓库同时承载内容策划、写作素材、学习笔记、研究文档，以及与内容生产相关的代码和产物。主题聚焦在 Agent skills、AI 工作流、AI 原生学习方式，以及一线实践中的方法、判断和踩坑经验。

## 项目目标

- 阶段目标：4 个月内全网粉丝达到 10000，总收入达到 10000。
- 持续输出 AI 实战内容，不做泛科普，不做空洞观点。
- 把复杂的 Agent skill 和 AI 工作流讲成人能直接拿去用的方法。
- 通过稳定更新，沉淀自己的内容资产、方法论和案例库。
- 用内容反向驱动研究、学习和产品化探索。

## 阶段指标

- 时间周期：4 个月。
- 粉丝目标：全网累计 10000。
- 收入目标：累计 10000。
- 内容节奏：每天 2-3 条，以视频为主，文字为辅。
- 核心策略：用高情绪浓度的实战内容换关注，用方法论产品化和服务化换收入。

## 内容定位

- 核心主题：Agent skills、AI 实战经验、工作流设计、学习方法、内容生产方法。
- 内容形态：视频 / 文字 = 2:1。
- 发布频率：每天 2-3 条内容。
- 内容来源：真实使用经验、研究笔记、工作流拆解、项目复盘、个人实验。

## 表达原则

这个项目最重要的不是“把信息讲清楚”，而是“让情绪先进去，再让认知留下来”。

- 情绪优先：先抓感受，再讲方法。
- 大字优先：标题、封面、关键信息都要能一眼打中人。
- 观点明确：不绕，不端着，不写中性废话。
- 经验导向：尽量来自真实操作、真实判断、真实问题。
- 可执行：内容最后要落到动作、方法或判断标准。

## 内容标准

- 选题要么有痛感，要么有反常识，要么有立刻能用的方法。
- 视频内容优先考虑口播张力、节奏和前三秒抓力。
- 文字内容优先考虑标题、结构和金句密度。
- 同一个主题尽量可以拆成视频版、文字版和延展素材。
- 每篇内容都应尽量回答一个具体问题，而不是泛泛表达态度。

## 仓库作用

这个仓库用于管理整条内容生产链路：

- 研究：收集 AI 相关实践、案例、方法和趋势。
- 学习：沉淀系统学习过程、诊断、课程、测验与复盘。
- 写作：保存选题、提纲、文章、口播稿和表达实验。
- 资产：归档图片、图示、草稿、产物和可复用模板。
- 工具：放置为内容生产服务的脚本、自动化和技能配置。

## 目录说明

- `posts/`：文章、口播稿、专题内容和相关素材。
- `research/`：研究报告、资料整理、趋势分析。
- `learning/`：学习计划、学习循环、测评和复盘记录。
- `notes/`：写作流程、方法备忘和日常思考。
- `skills/`：与 Agent skills 或内容生产相关的能力配置。
- `tasks/`：任务拆解、执行计划和流程性事项。

## 生产原则

- 一个主题，优先思考能不能拆成 3 条以上内容。
- 先有钩子，再有结构，再补证据。
- 先写能打人的一句话，再展开方法。
- 优先复用已有研究和学习材料，减少重复劳动。
- 所有内容尽量形成可检索、可复盘、可重组的资产。

## 当前方向

现阶段内容重点：

- 分享 Agent skills 的设计、使用方式和实战案例。
- 分享 AI 在学习、写作、研究和工作流里的具体应用。
- 分享一线使用 AI 时形成的判断、经验和方法论。
- 把抽象能力拆成普通人可以直接照做的步骤。

## 一句话定义

这是一个把 AI 实战经验做成高频内容、把内容沉淀成长期资产的仓库。
