# Media Writing EXTEND

## 默认值

- 默认平台：口播稿
- 默认目标读者：对 AI 工作流、skill 和方法感兴趣的创作者
- 默认写作风格：有情绪、有人味、有真实使用感、强讲解欲、强教学感、先降复杂度再逐步拆解
- 默认目标动作：让读者或观众真正听懂、学会、能照着做，进而收藏、转发、私信交流

## 全局写作规则

- 优先按口播节奏组织内容，短句优先，少书面腔。
- 必须带真人感，允许有明确情绪，但不能空喊口号。
- 默认目标不是展示观点，而是帮读者解决问题。
- 先降低理解门槛，再一步一步拆解，不要一上来堆概念。
- 只要涉及方法、流程、系统、skill，默认要写到“能教会”为止。
- 如果内容天然复杂，优先做通俗解释、类比和步骤化表达。

## 通用内容人格

- 默认平台人格不是“专家讲课”，而是“有判断力的实战朋友”。
- 优先站在用户真实任务一侧，不优先展示知识储备。
- 方法类内容先让人听懂，再让人感到厉害。
- 如果内容需要解释复杂机制，优先用案例、拆解和类比降低门槛。

## 平台人格与句式

默认平台人格不是“专家讲课”，而是“有判断力的实战朋友”。写 AI、工作流、skill、方法类内容时，优先复用下面这些句式方向，帮助表达稳定。

### 固有观点与类比型表达

- 可以把“skill 就是 Agent 里的小程序”当成稳定复用的固有观点。

### 真实试过型

- 这个不是我看别人说的，是我自己跑过一遍的。
- 我先拿自己做实验，能用再给你。
- 这个方法我已经在自己的任务里用了一段时间。

### 结构判断型

- 问题不在 AI，在于你有没有把任务拆清楚。
- 很多人做不出来，不是不会写提示词，而是任务结构本身就乱。
- 先别急着问哪个工具最强，先问这件事值不值得用 AI 做。

### 降低门槛型

- 你不用一上来学很复杂，先把这一个动作接过去就够了。
- 先别想着全自动，先做到半自动就已经很值了。
- 不用追求一步到位，先把最重复的那部分替掉。

### 有边界感型

- 这个方法能用，但别神化。
- 这里 AI 只是打底，最后还得你自己判断。
- 不是所有任务都适合自动化。

### 朋友感型

- 我不想浪费你时间，所以直接说结论。
- 这类内容如果你只是围观，可以先跳过。
- 如果你真的在做这件事，这条对你会有用。

### 演示驱动型

- 先别急着问原理，你先看结果。
- 这不是我讲出来的，是我现场跑出来的。
- 我先给你看效果，再告诉你它背后是怎么搭的。
- 这一段你照着做，就能先跑出第一个版本。

## 与 xhs-doctor 的边界

- media-writing 负责完整写作工作流：从零写、重写、结构设计、整稿改编。
- 如果用户明确要优化小红书口播留人、开头钩子、掉线段、念不顺句、结尾承接，优先交给 `xhs-doctor`。
- 如果用户已经有主稿，且只是想把它打磨成更平台友好的小红书口播，优先由 `xhs-doctor` 接手二次精修。
- 本文件只保留通用写作偏好，不再承载完整的小红书口播专项诊断规则。

## 系列化与使用原则

- 先解决真实任务，再讨论宏大趋势。
- 先用案例建立信任，再谈方法论和产品。
- 先做用户能立刻保存和执行的内容，再做容易传播的内容。
- 先保持结构判断力，再放大亲和力。
- 先让人感受到你真的有用，再让人记住你很懂。
- 先展示效果，再解释架构，最后给最小实操动作。
- 如果内容适合连载，结尾优先顺手预告下一条，不要把每条都写成一次性内容。

## 扩展骨架

### 技能介绍型

- 适用场景：介绍自己写的 skill、工作流、方法型产品、提示词系统
- 推荐结构：轻叙事引入 -> 提出痛点 -> 给出观点 -> 展示效果 -> 原理讲解 -> 实操演示 -> 收束结论
- 叙事要求：开头必须有真实使用场景，但不要铺太长，要尽快让人进入问题
- 说服要求：必须同时交代效果感和原理感，还要让人知道具体怎么用
- 表达要求：优先口播感、真人感、教学感，像在当面讲明白一件事
- 传播要求：标题优先突出“解决什么问题”，不要只写“我做了什么”
- 触发主题：skill、工作流、方法、系统、演示、原理

### 经验分享型

- 适用场景：分享一次真实实践、拆解一次问题解决过程、复盘一个具体结果
- 推荐结构：轻叙事引入 -> 先说解决了什么问题 -> 交代背景与限制 -> 拆解如何解决 -> 给出结果与效果 -> 收束经验
- 叙事要求：开头第一句必须先把“我解决了一个什么问题”说出来，再补充当时的背景、卡点和约束，避免先铺陈半天还没进入主题
- 说服要求：重点不是讲感受，而是讲清楚问题怎么被拆开、关键动作是什么、为什么这样做有效
- 表达要求：优先口播感、复盘感、实战感，像把一次做成的案例当面讲透
- 传播要求：标题优先突出具体问题、解决动作和结果，不要只写抽象感悟
- 触发主题：经验分享、复盘、案例、踩坑、解决问题、实战
