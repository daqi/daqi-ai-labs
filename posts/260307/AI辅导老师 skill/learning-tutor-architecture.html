Skill Architecture

Learning Tutor 不是课程流水线，而是一个回合级导师引擎

这个 skill 的第一抽象不是“课程步骤”，而是“学习者当前状态 + 下一拍教学动作”。它围绕会话加载、状态判断、教学动作调度、证据写入和长期 learner model 演化运转，核心价值来自实时干预，而不是一次性讲完。

Session Bootstrap Tutor Orchestrator Intervention Engine Assessment + Review Learner Memory
Primary Unit

1 回合

每次只盯一个焦点 concept 或 skill，不把 session 变成大课件。

Core Asset

Learner Model

系统沉淀“这个人怎么学、常错什么、该怎么教”，不是只记内容进度。

Control Logic

State Machine

在 diagnose、teach、drill、remediate、review 间动态切换。

Persistence

文件协议

topic 级长期记忆与 session 级证据记录分层存放，支持无缝续学。

与 learning-loop 的边界

learning-loop 偏流程型学习 SOP；learning-tutor 偏真人老师式互动。前者按轮生成学习流程，后者按回答实时追问、纠错、降维和切换讲法。

关键判定信号

系统不会把“用户说懂了”当作懂了，而是通过 probe、practice、transfer 暴露证据，再决定升级、降维或进入 remediate。

架构上的真正闭环

用户回答不是终点，而是下一动作的输入；而每轮证据又会反过来更新 meta、learner-model、misconceptions 与下次 review 计划。

2. Runtime

运行时模块拆解

如果把整个 skill 看成一个在线老师，这几块分别回答：先读什么、先盯什么、怎么问、怎么纠错、怎么记住这个学生。

Bootstrap

Session Bootstrap

负责新主题初始化、继续上次 session、切 review 模式，以及从历史文件中恢复当前上下文。

  • 判断 diagnose / teach / drill / remediate / review
  • 锁定 active_session、current_mode、current_focus
  • 把 topic 级长期记忆交给调度器
Control Core

Tutor Orchestrator

这是 skill 的大脑。它不是线性流程，而是一个会回跳的状态机：先判断学员状态，再决定下一拍该讲、该问、还是该修误解。

  • 选择当前 focus concept 或 skill
  • 判断教学动作是否失效并切换策略
  • 在 NEXT_DECISION 中决定 continue / remediate / switch / close
Intervention

教学动作引擎

把“知道用户答错了”翻译成“该怎么教”。系统显式区分 probe、hint、contrast、counterexample、reframe、transfer 等动作。

  • 同一低效动作不能连续硬顶
  • 连续两轮失败必须降维
  • 误解稳定复发则升级到 remediate
Assessment

证据与错误类型识别

收到回答后先分类再回应，避免一错就把标准答案端出来。真正记录的是 evidence，而不是模糊印象。

  • 识别 concept-confusion、boundary-failure、transfer-failure 等错误
  • 也识别 accurate-transfer、creative-link 等正向信号
  • 用 evidence 推动 mastery、confidence 与 next action
Memory

Learner Model

存的是“这个人怎样学”而不是“这个主题讲到哪”。这让 tutor 在第二次、第三次继续时，越来越像认识这个学生。

  • 全局层：动机、应用场景、目标深度、自评偏差
  • 概念层：mastery、confidence、常见错误、misconception IDs
  • 行为层：提示阈值、节奏偏好、讲解偏好
Knowledge

Domain Model + Review Planner

Domain Model 把主题拆成可诊断的最小单元，Review Planner 决定下次优先抽查哪个脆弱点，形成长期维护闭环。

  • 主题拆成 concept / relation / skill
  • 判断卡点是前置缺失还是迁移失败
  • 复习模式优先打最脆弱且最关键的点
3. Loop

回合教学循环

单轮交互的正确颗粒度是“一个焦点 + 一个动作 + 一个可判断的回答”。这也是它和一次性长文讲解最根本的分界线。

01

选焦点

优先用户明确卡住的点，其次历史最高风险误解，再其次 learner model 中关键且脆弱的 concept。

02

选动作

根据当前 mode 与回答状态，在 probe、teach、hint、contrast、practice、transfer 中选一个最有效动作。

03

收回答

用户回答不是被动接收内容，而是暴露真实掌握度、自评偏差和表达断点的关键证据。

04

识别错误

先判断是 definition-gap、causal-break、retrieval-failure，还是准确迁移与创造性链接，再决定下一步。

05

写入证据

evidence 进入 transcript、evidence、assessment，并反向更新 meta、learner-model、misconceptions 与 review_state。

退出条件同样是架构的一部分：当前点已过关、收益开始下降、或识别出下次更适合单独开新 session 时，系统才结束本次 tutor session。
4. Persistence

记忆与文件协议

这个 skill 的“长期性”来自显式文件协议。topic 级目录保存长期画像，sessionN 保存单次交互证据；二者分层，才支持后续继续学而不是每次重新摸底。

Topic Session Root

目录结构

Global State

meta + journal

meta.md 提供当前 mode、focus、overall mastery、review_state。journal.md 以追加式方式记录每次 session 摘要，避免上下文丢失。

Knowledge Shape

domain-model

把 topic 从一句话拆成 concept、relation、skill，给调度器一个可诊断、可教学、可评估的对象集合。

Human Memory

learner-model + misconceptions

前者记录长期掌握与偏好，后者记录稳定误解及其 active/watch/resolved 状态，是 remediate 的直接输入。

Session Evidence

plan / transcript / evidence / assessment

把单次 session 的计划、关键回合摘要、证据条目和收尾总结拆开，既能复盘，也能喂回长期模型。

5. Entry Points

脚本与操作入口

实现层没有把所有逻辑藏进 prompt，而是显式提供脚手架脚本和 payload 模板。这样 skill 的行为既可自动调度，也可被可靠地写回文件系统。

Bootstrap Script

init_session.py

负责 topic 级目录存在性、基础文件缺省、创建新的 sessionN 脚手架，以及返回当前状态摘要。

action=init action=status action=start-session 
python3 skills/learning-tutor/scripts/init_session.py \
  --topic "主题" \
  --sessions-dir learning/tutor-sessions \
  --action start-session \
  --mode teach
Update Script

update_session.py

负责追加 transcript、写 evidence、收尾 assessment，并同步更新 learner-model、misconceptions、journal 与 meta。

append-round finalize-session payload-file 
python3 skills/learning-tutor/scripts/update_session.py \
  --topic "主题" \
  --sessions-dir learning/tutor-sessions \
  --action finalize-session \
  --payload-file /tmp/finalize-session.json
Payload Templates

机器可写输入协议

append-round.json 和 finalize-session.json 让状态更新不靠临时口头约定字段名，而是落到稳定的写入契约。

  • round 级写入 transcript 与 evidence
  • session 级写入 assessment 与 journal 摘要
  • concept_updates / misconception_updates 驱动长期状态变化
Architecture Reading

设计上的一条底线

这个 skill 成功与否，不取决于一次输出多漂亮，而取决于它能否在多次继续学习中越来越理解“同一个人”的掌握状态、误解模式和节奏偏好。

  • 不把用户自评直接当真
  • 不为了顺滑而跳过逻辑断点
  • 不为覆盖面牺牲误解修复
