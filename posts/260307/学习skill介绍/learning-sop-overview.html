Adaptive Learning Workflow

Learning SOP 架构示意图

这是一个以“诊断 → 生成课程 → 学习互动 → 测评决策”为核心闭环的 AI 学习工作流。它不是一次性课程，而是会根据当前水平、遗忘衰减和复习表现持续调整难度与路径。

01

一轮学习循环

流程的核心不是“讲课”，而是先校准水平，再只补盲区，最后依据表现决定继续提升还是切换到复习维护。

Core Loop

Learning SOP 的最小工作单元是一轮循环。每轮都从当前真实水平出发，先做诊断，再生成定制课程，随后通过互动和测评验证掌握，而不是把“完成阅读”误当成“学会了”。

输入 topic、cycle、resume/status 指令，以及历史 session 状态。
中间产物 诊断、课程、答题记录、评估四份文件，支撑下一轮上下文。
输出 新的 knowledge_level、下轮 blind spots、或者 review 计划。
Mastery Rule

综合掌握度会被折算成新的 knowledge_level，单轮提升最多 +2，避免因短期表现好而高估掌握水平。

未达标 new_level < 8，继续进入下一轮学习。
达标 new_level ≥ 8，停止新课循环，切换到 SM-2 复习模式。
Step A

水平诊断

diagnosis.md

首轮通过背景问答定级，后续轮次会按距离上次学习的时间做遗忘衰减，并用热身检索校准实际记忆状态。

  • 第 1 轮问 2-3 个问题定位水平与动机。
  • N>1 时读取上轮 assessment 与 meta 计算 decay。
  • 间隔 ≥ 3 天，触发热身检索确认“记得多少”。
Step B

按需生成课程

curriculum.md

课程只围绕盲区展开，用已掌握概念当锚点，不重复用户已经会的内容，并为后续互动预埋练习。

  • 根据 knowledge_level 选择入门到专家难度带。
  • 每个模块只对应一个 blind spot。
  • 课程自包含，不依赖外部资料。
Step D

测评与决策

assessment.md

先做元认知自评，再结合三段互动结果计算综合掌握度，更新 knowledge_level，并决定继续学习还是切到 review。

  • 综合分 = 费曼 40% + 检索 30% + 迁移 30%。
  • delta 被限制在 -1 到 +2 之间。
  • 达标后写入 next_review 与 efactor。
Step C

学习互动

quiz.md

通过三段互动检验“会不会说、能不能想起、是否能迁移”，避免课程读完但没有形成可调用知识。

  • C1 费曼测试：暴露理解漏洞。
  • C2 检索练习：重写新题，不能直接复用课程题。
  • C3 迁移应用：在新场景验证抽象能力。
Cycle N 诊断当前水平，补盲区，测掌握，再决定是否升阶。
Branch 1

继续下一轮

当 new_level 小于 8 时，系统保留剩余盲区，更新 current_cycle = N+1，再从 Step A 重新开始，但使用新的上下文与更高难度。

Branch 2

切换复习模式

当 new_level 大于等于 8 时，工作流停止“新课生成”，改为按 next_review 节奏执行检索复习与 Step D 维护记忆稳定性。

02

运行入口与分流

用户触发的其实不是单一路径。不同参数会把流程带到初始化、继续学习、查看状态或跳轮加载等不同入口。

Command Surface
/learning-loop topic="X"

新建或继续

默认读取现有 session；如果不存在则初始化，knowledge_level 从 0，current_cycle 从 1 开始。

action=resume

恢复中断

优先从 meta 里的 current_cycle 与 mode 继续，不需要用户手动判断自己停在第几轮。

action=status

只看进度

读取 meta 并打印摘要后立即停止，不进入 Step A/B/C/D，适合查看当前掌握状态和 review 日期。

cycle=N

跳轮进入

强制从第 N 轮开始，但会先读取 N-1 轮 assessment，避免跳轮后上下文断裂。

初始化脚本 init_session.py 负责创建或读取 session 目录，输出 current_cycle、knowledge_level、mode、efactor、next_review 等状态给后续流程使用。
03

记忆文件结构

这个技能的关键资产不是 prompt 本身，而是会持续累积的 session 记忆。没有这些文件，就没有真正的自适应学习。

Session Storage
learning/sessions/{slug}/ ├── meta.md # 状态总表 ├── journal.md # 轮次摘要 + 用户自由记录 └── cycles/ ├── cycle1/ │ ├── diagnosis.md # Step A 水平诊断结果（含遗忘衰减） │ ├── curriculum.md # Step B 本轮生成的课程 │ ├── quiz.md # Step C C2 作答记录 │ └── assessment.md # Step D 综合评估 + 循环决策 ├── cycle2/ │ └── ... └── cycleN/
meta.md

meta 是整个 session 的控制台，存放 topic、current_cycle、knowledge_level、mode、efactor、next_review、review_count 等全局状态。

current_cycle 指向下一次要执行的轮次，而不是上一次完成的轮次。
knowledge_level 0-10 的长期掌握估计，会被遗忘曲线和测评共同影响。
mode 只分 learning 与 review，两者决定执行路径完全不同。
journal.md

journal 是用户自由区。AI 只能在末尾追加本轮摘要，不能覆盖已有记录，保证人的长期思考也能留在同一个 session 中。

04

适应规则与算法

这部分决定了它为什么不是普通课程生成器。真正的自适应来自两类机制：学习轮之间的遗忘校正，以及达标后的复习调度。

Adaptive Logic
Forgetting Curve
间隔 衰减 动作
小于 1 天 0 视为连续学习，不做衰减。
1-2 天 -0.5 轻微遗忘，下轮仍可直接衔接。
3-6 天 -1.0 触发热身检索，确认真实水平。
7-13 天 -1.5 盲区可能回流，需要更强校准。
14-29 天 -2.0 大量遗忘，本轮课程会明显回补。
30 天及以上 -2.5 接近重学状态，必要时回到更基础解释。
SM-2 Review
复习表现 q E-Factor 结果
5 +0.1 秒答，间隔正常增长。
4 不变 记得较稳，继续扩间隔。
3 -0.14 模糊记得，间隔仍增长但更保守。
2 -0.30 大部分忘记，interval 重置为 1。
0-1 -0.50 严重遗忘，必要时触发完整循环重学。
05

状态迁移视角

从系统设计角度看，Learning SOP 更像一个状态机。不同模式读取不同上下文，也写入不同类型的长期记忆。

State Model
State 1

Session 初始化

通过 init_session.py 创建 slug 目录、meta 和 journal。此时 knowledge_level = 0，mode = learning，工作流还没有任何 cycle 文件。

State 2

Learning 循环中

系统围绕 current_cycle 执行 A/B/C/D，并把每轮结果追加进 cycles。只要 new_level 小于 8，就一直留在这个状态。

State 3

Review 维护中

一旦进入 review，就不再默认生成新课程，而是基于 next_review 和 efactor 做检索复习；若复习表现过差，状态机会把用户送回完整学习循环。

这也是 learning-loop 和常见“生成一份课程大纲”型技能的本质差异：它有长期状态、有可回放的轮次档案、有节律控制，因此能做真正的渐进式掌握，而不是一次性输出内容。
