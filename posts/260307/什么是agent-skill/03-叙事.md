# 03 叙事

## 候选开场场景

- 场景 A：很多人都在讲 skill，讲了半天你还是没懂。因为他们讲的都是定义，我这次不讲定义，我直接把一个我最近真的在跑的 learning-loop 拿出来给你看。
- 场景 B：你打开一个 agent，输入一个学习主题，它先判断你水平，再生成课程，再互动，再测评，还会写文件、记进度。你看到这里，才会第一次意识到，原来 skill 不是一段提示词，它更像一个挂在 agent 身上的小应用。
- 场景 C：如果我只把 learning-loop 的 SKILL.md 丢给你，你大概率还是看不懂。但如果我先让它跑起来，再倒回去拆 name、description、语法、工作流、记忆系统，你一下就明白了。

## 推荐叙事抓手

- 推荐开场：场景 A + 场景 B 组合
- 推荐理由：A 先点破观众的真实困境，B 立刻给动作画面。对口播来说，这样进入最快，不需要长铺垫，也能自然过渡到演示。
- 预计带出的核心问题：为什么很多人讲过 skill，你还是没真正理解；以及 skill 到底为什么必须叫 agent skill

## 核心冲突

- 表层冲突：大家以为自己没听懂 skill，是因为概念太复杂；其实不是，真正的问题是别人一直在讲定义，没有先让你看到它怎么工作。
- 深层冲突：skill 看起来像一个文档或一个目录，但它真正的价值来自 agent 的执行能力。脱离 agent，它只是说明书；挂到 agent 上，它才会调用工具、组织流程、读写文件、产生结果。
- 为什么这个冲突值得读：因为你的目标读者不是第一次听说 skill，而是已经接触过 agent，却还没把“文档”“提示词”“工作流”“工具调用”这些东西在脑子里连成一个整体。

## 关键变化

- 变化前：听众对 skill 的理解是模糊的，可能把它当成一段高级提示词，或者一个写法更复杂的模板。
- 转折点：我不继续抽象解释，而是直接拿 learning-loop 演示它如何被调用、如何执行、如何把结果写进文件和记忆。
- 变化后：听众意识到 agent skill 的本质不是“写了一份说明”，而是“封装了一套 agent 可以反复调用的工作能力”。

## 可复用细节

- 具体瞬间：用户给出一个学习主题后，learning-loop 先初始化 session，创建目录和脑图，再按诊断、课程、互动、测评的顺序推进，并把结果写回对应文件。
- 对话/原话：很多人都讲过 skill，但你可能还是不理解。不是你笨，是他们一直在解释名词，没有先让你看到它怎么干活。
- 动作或反应：先跑一次 learning-loop，再回头拆 SKILL.md、assets、references、scripts，听众会从“这是什么”切到“原来它是这么组织起来的”。
- 数字或事实细节：learning-loop 明确写了命令语法、四步循环、session 目录结构，以及按轮次写入 diagnosis、curriculum、quiz、assessment 等文件。

## 待用户确认

- 最想保留的场景：推荐保留 A + B 组合
- 冲突是否准确：推荐保留“别人老在讲定义，没有先演示”这条冲突
- 还需要补什么具体细节：如果你愿意，我下一步可以在说服层里把 learning-loop 的几个关键事实再钉得更具体，比如命令格式、四步链路、写入哪些文件
