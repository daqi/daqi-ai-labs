## 主稿

- 标题：什么是 agent skill？我用一个真实案例，3 分钟给你讲明白
- 开头：我不跟你空讲什么叫 skill，我直接把我最近做的一个 skill 跑给你看。你看完这个过程，就知道为什么它应该叫 agent skill。
- 正文：

  很多人都讲过 skill，但你可能还是不理解。

  不是你没听懂。

  是他们老在解释定义，没有先让你看到它怎么干活。

  所以这次我不空讲概念，我直接拿一个我最近做的交互式学习 skill，叫 learning-loop，给你拆。

  你把它想成什么？

  你可以先把它想成 Agent 里的微信小程序。

  为什么我不用别的类比？

  因为这个类比最接近它的真实感觉。

  你在微信里点开一个小程序，它不是给你看一段说明，它是直接开始提供一个具体能力。

  agent skill 也是一样。

  它不是一段更复杂的提示词，也不是一个摆在那里的模板文件。

  它是一个挂在 agent 身上的能力单元。

  到了该触发的时候，agent 会把它调起来，然后它开始按自己的流程去干活。

  我拿 learning-loop 举例。

  当用户说，我想系统学习一个主题，比如奥派经济学，或者他说从第 2 轮开始，或者我想看进度，这个 skill 就会被触发。

  一旦触发，它不是回你一段解释就结束了。

  它会先初始化一个 session，创建目录，准备脑图文件。

  然后它会判断你从哪一轮开始。

  接着进入一整套学习循环。

  Step A 先诊断你的水平。

  Step B 按你的盲区生成课程。

  Step C 跟你做学习互动。

  Step D 再做测评和下一轮决策。

  这还没完。

  它会把每一轮的结果写进对应文件里。

  比如 diagnosis、curriculum、quiz、assessment。

  它还会更新 meta.md、journal.md，记录你现在学到第几轮，知识水平大概在哪，下一次什么时候复习。

  你看到这里，就该明白一件事了。

  这已经不是一段提示词了。

  这是一套可以被 agent 调起来、还能持续运行、还能把状态写回去的能力封装。

  这就是为什么我说，skill 应该叫 agent skill。

  因为脱离 agent，它根本跑不起来。

  你单独把 learning-loop 的 SKILL.md 拿出来看，它当然重要，但它本身不会执行。

  真正让它活起来的，是 agent 看到了它的 description，知道在什么场景下该调用它；看到了它的语法，知道该怎么把用户输入喂给它；再按照它定义的工作流，去读 references、调 scripts、写文件、更新记忆。

  所以你会发现，一个 skill 的关键，不在于你文件夹摆得多漂亮。

  而在于这几个东西有没有接上。

  第一，名字和 description。

  这决定 agent 知不知道什么时候该用它。

  第二，语法和输入输出。

  这决定它接任务的时候会不会接错。

  第三，工作流程。

  这决定它不是随便发挥，而是按一套可复用的步骤往下走。

  第四，记忆系统。

  这决定它是不是一次性回答，还是一个有状态、能延续的能力。

  第五，assets、references、scripts 这些资源层。

  这决定它有没有外部支撑，能不能把复杂任务拆开做，而不是全靠模型当场现编。

  这时候很多人会反驳一句。

  他说，这不还是提示词吗？只是写得更复杂一点。

  不是。

  更准确地说，不是所有写得长的东西都叫 skill。

  也不是所有 skill 都必须很复杂。

  判断一个东西是不是 skill，不看它有多少文件，不看它目录有多满，而看它能不能在一个明确场景里，被 agent 稳定触发，接管任务，调动资源，然后产出结果。

  轻一点的 skill，可能只有一个 SKILL.md。

  重一点的 skill，可能还会带 scripts、templates、references、评估链路。

  但它们的本质是一样的。

  它们都不是静态资料。

  它们都是 agent 可调用的工作单元。

  所以如果你准备开始自己写 skill，我建议你先别上来就琢磨怎么把文案写得高级。

  先问自己 4 个问题。

  第一，这个 skill 到底在什么场景下触发。

  第二，它接到任务之后，第一步到最后一步怎么走。

  第三，它要调用什么工具、什么文件、什么参考资料。

  第四，它做完之后，结果落到哪里，状态怎么延续。

  你把这 4 件事想明白，再回头写 SKILL.md，很多东西就顺了。

  不然你大概率写出来的不是 skill。

  只是一个说明书。
- 结尾：所以，什么是 agent skill？你可以先记住一个最容易用的理解。skill 就像 Agent 里的微信小程序。它不是放在那里给你看的，它是到了合适场景就会被点开、被调用、开始干活的。你一旦这样理解，后面再看 name、description、语法、工作流、记忆系统、assets、references、scripts，这些东西就全串起来了。
